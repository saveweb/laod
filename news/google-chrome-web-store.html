谷歌将禁止从第三方来源安装Chrome扩展程序（插件）

2018-06-14 12:13 • 业界资讯 • 阅读 8993

长久以来，Google 一直在努力尝试提升 Chromium 系列网络浏览器的扩展安装体验。除了允许用户通过 Chrome Web Store 获取之外，不希望公开发布的开发者也可以通过第三方来安装。不过今后，事情可能会发生一些变化。Google 在本周二宣布，该公司计划撤掉 Chrome 浏览器扩展的“内联式安装”（inline installations）功能，意味着用户将无法再通过第三方网站安装它们。

之所以如此突然地宣布要封杀第三方安装 Chrome 扩展的选项，是因为之前 Google 收到了许多起用户被欺骗的报告。

扩展平台产品经理 James Wagner 解释到：

在 2018 年 6 月 12 号之后首发的扩展，当其尝试调用 chrome.webstore.install() 函数的时候，用户将被重定向至新标签页上打开的 Chrome Web Store 以完成安装。

今年夏末（2018 年 9 月 12 日起），现有扩展的内联安装选项也将被禁用。

届时如果用户还想通过第三方网站安装扩展，也会被跳转至 Chrome 网上应用店。

最终，Google 计划在 2018 年 12 月上旬撤掉内联安装 API：

Chrome 71 将会是首个这么做的浏览器版本，至于那些通过自家网站发布扩展的者，Google 建议他们将链接重定向到 Chrome Web Store 。

那些不能再 Chrome Web Store 发布、或者因为各种原因而不这么做的，也将没有第二条路可以选 —— 这些原因与 Google 的政策无关，他们只能抱憾退出。

Google 表示，此举将让用户在未来受益，使 Chrome 浏览器变成满足其浏览需求的更安全之地。

必须通过 Chrome store 安装扩展， 等同于 开发 Chrome 扩展必须认证开发者，相当于 $5 / developer(new) . 这波有得赚。

另，强制从 Chrome store 安装扩展， 对于用户来说，并不能阻止恶意扩展。对于中国用户来说，就呵呵了

最后提一下，Chrome 扩展的开发者认证，国家一栏，是没有中国这个选项的。
